## 简介

欢迎使用豆匣。本手册按操作目标组织，引导您了解每一项功能。豆匣为单用户桌面应用，所有数据均保存在您的本机。

## 入门指南

### 任务：安装豆匣

**前提条件：**
- Windows 10+ 或 macOS 11+（Apple Silicon 或 Intel）
- 下载需要联网

**步骤：**

1. 从 [GitHub Releases 页面](https://github.com/KylinLabAI/DouXia-App/releases/latest) 下载安装包
   - Windows：`DouXia-Setup-x.x.x.exe`
   - macOS（Apple Silicon）：`DouXia-x.x.x-arm64.dmg`
   - macOS（Intel）：`DouXia-x.x.x-x64.dmg`
2. 运行安装程序并按提示操作
3. 从应用程序文件夹启动豆匣
   - 您将看到：包含 对话、知识、设置 三个标签页的主窗口

**预期结果：** 豆匣启动并显示主界面。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| Windows 拦截安装包 | 右键文件 → 属性 → 勾选“解除锁定”后重新运行 |
| macOS 提示“无法打开” | 右键 → 打开，或在 系统设置 → 隐私与安全性 中允许 |

### 任务：首次启动与平台登录

**前提条件：**
- 已安装豆匣

**步骤：**

1. 打开“对话”标签页
2. 选择偏好的 AI 平台（豆包、元宝、DeepSeek 或 ChatGPT）
3. 在内嵌浏览器中登录您的 AI 账号
   - 您将看到：熟悉的 AI 平台界面显示在豆匣中
4. 开始对话——默认自动捕获对话内容
   - 您将看到：捕获指示器确认对话正在保存

**预期结果：** 可与 AI 平台对话，且对话自动捕获到本地数据库。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 平台页面无法加载 | 检查网络连接；内嵌浏览器需要联网 |
| 对话未被捕获 | 在 设置 → AI 平台 中确认已开启自动捕获 |

## 日常操作

### 任务：导入已有对话

**前提条件：**
- 在源 AI 平台拥有账号
- 已在豆匣“对话”标签页登录该平台

**步骤：**

1. 进入“知识”标签页
2. 点击“导入”按钮
   - 您将看到：导入对话框
3. 选择要导入的来源平台
4. 选择导入范围（全部对话或仅近期对话）
5. 确认并等待导入完成
   - 您将看到：导入的对话出现在知识列表中

**预期结果：** 历史对话被加入本地知识库。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 导入速度慢 | 历史记录较多时需要时间，请保持窗口开启直到完成 |
| 部分对话缺失 | 重新运行导入；过旧的对话可能需要先手动滚动加载 |

### 任务：搜索与筛选对话

**前提条件：**
- 已导入或已捕获对话

**步骤：**

1. 进入“知识”标签页
2. 在搜索栏输入关键词进行全文搜索（FTS5）
   - 您将看到：跨所有平台的匹配对话
3. 点击“筛选”按钮缩小结果范围
   - 按平台、日期范围、标签或状态筛选
4. 点击某条对话查看完整内容

**预期结果：** 可快速定位跨所有平台的任意对话。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 搜索无结果 | 检查拼写或放宽日期范围筛选 |
| 筛选不生效 | 刷新“知识”标签页后重试 |

### 任务：使用标签、聚类与图谱整理

**前提条件：**
- 知识库中已有对话
- 已配置 AI 服务（设置 → AI 配置）

**步骤：**

1. 选中一条或多条对话，触发 AI 自动打标签
   - 您将看到：建议的话题标签被添加到对话
2. 打开“话题聚类”查看自动分组的对话
   - 点击某个聚类查看其中所有对话
   - 可重命名聚类以便整理
3. 打开“知识图谱”可视化话题之间的联系
   - 点击节点查看相关对话
   - 使用缩放控件进行探索

**预期结果：** 对话被组织为可导航、可视化的知识结构。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| AI 打标签未生成标签 | 在 设置 → AI 配置 中确认已设置 AI API Key |
| 图谱为空 | 请先生成标签和聚类；图谱依赖已标记的话题 |

### 任务：AI 摘要与知识卡片

**前提条件：**
- 在“知识”标签页已选中对话
- 已配置 AI 服务

**步骤：**

1. 选中一条对话并触发 AI 摘要
   - 您将看到：一段精炼概述保存在对话旁
2. 如需提取结构化笔记，触发 AI 笔记（知识卡片）
   - 您将看到：抽取出的关键点与可执行项
3. 审阅并编辑生成的内容
4. 将知识卡片保存到知识库

**预期结果：** 长对话被提炼为可复用的摘要和结构化笔记。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 摘要不准确 | 保存前手动编辑结果 |
| AI 请求失败 | 检查 API Key、模型参数及网络连接 |

### 任务：AI 推荐清理

**前提条件：**
- 知识库中有多条对话
- 已配置 AI 服务

**步骤：**

1. 打开 AI 推荐删除
   - 您将看到：AI 认为过时或低价值的对话列表
2. 审阅推荐结果
3. 选择要删除的对话并批量删除
   - 您将看到：所选对话被永久删除

**预期结果：** 知识库被精简，仅保留有价值的对话。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 有用对话被标记 | 删除前取消勾选；推荐仅供参考 |
| 批量删除较慢 | 选择较多时需要时间，请等待确认后再关闭 |

### 任务：删除对话

**前提条件：**
- “知识”标签页中存在对话

**步骤：**

1. 选中一条或多条对话
2. 点击删除
   - 您将看到：确认提示
3. 确认删除
   - 您将看到：对话从列表中移除

**预期结果：** 不需要的对话被永久删除。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 误删对话 | 从最近的数据库备份恢复（设置 → 数据文件夹） |

### 任务：导出对话

**前提条件：**
- 有需要导出的对话
- （可选）已在 设置 → 文件导出文件夹 中设置默认导出目录

**步骤：**

1. 选中一条对话或一个集合
2. 点击导出并选择格式：Markdown、HTML 或 PDF
3. 选择目标文件夹（或使用默认目录）
   - 您将看到：导出文件保存到所选位置

**预期结果：** 对话以所选格式导出。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 导出失败 | 确认目标文件夹可写且磁盘空间充足 |
| PDF 导出内容缺失 | 改用 HTML 或 Markdown 格式作为替代 |

### 任务：局域网分享知识库

**前提条件：**
- 同一局域网内有其他设备
- 查看设备上装有网页浏览器

**步骤：**

1. 进入 设置 → 局域网分享
2. 启用局域网分享并设置端口号
   - 您将看到：显示一个可分享的 URL
3. 在其他设备的浏览器中打开该 URL
   - 您将看到：其他设备可浏览您的知识库

**预期结果：** 局域网内其他设备可通过浏览器浏览您的知识库。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 其他设备无法连接 | 确认两台设备在同一网络，且端口未被防火墙拦截 |
| URL 未显示 | 重新启用局域网分享以重新生成 URL |

## 设置指南

### 任务：配置 AI 服务

**前提条件：**
- 拥有受支持 AI 服务商的 API Key

**步骤：**

1. 进入 设置 → AI 配置
2. 输入您的 API Key
3. 选择 AI 服务商
4. 配置模型参数（温度、最大 token 数）
5. 保存设置并验证连接
   - 您将看到：绿色状态指示器确认 AI 服务可达

**预期结果：** 智能功能（打标签、摘要、聚类）被启用。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| AI 服务不可达 | 核实 API Key 与网络访问 |
| 结果质量低 | 调整温度或切换到其他模型 |

### 任务：管理 AI 平台

**步骤：**

1. 进入 设置 → AI 平台
2. 启用或禁用特定 AI 平台
3. 配置平台专属设置
4. 设置新会话的默认平台
   - 您将看到：平台列表按您的选择更新

**预期结果：** 仅显示您偏好的平台，并已选定默认平台。

### 任务：设置数据与导出文件夹

**步骤：**

1. 进入 设置 → 数据文件夹
   - 选择 SQLite 数据库的存放目录
   - 查看数据库大小与存储占用
   - 如需可移动数据库到其他位置
2. 进入 设置 → 文件导出文件夹
   - 设置默认导出文件夹
   - 选择默认导出格式与选项

**预期结果：** 数据与导出文件存放在您选择的位置。

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 数据库移动失败 | 确认豆匣处于空闲状态且目标文件夹可写 |

### 任务：配置自动导入

**步骤：**

1. 进入 设置 → 自动导入
2. 启用自动导入
3. 设置导入频率
4. 选择要自动导入的平台
   - 您将看到：导入计划已激活

**预期结果：** 新对话按设定计划自动导入。

### 任务：切换语言

**步骤：**

1. 进入 设置 → 语言
2. 选择英文或中文
   - 您将看到：界面立即更新

**预期结果：** 所有界面文本与菜单完成翻译。

## 故障排查

### 任务：数据库备份与恢复

**步骤：**

1. 进入 设置 → 数据文件夹
2. 使用备份选项创建数据库快照
3. 如需恢复，选择一个之前的备份文件
   - 您将看到：数据库恢复到快照状态

**故障排查：**

| 问题 | 解决方案 |
|------|----------|
| 恢复失败 | 确认备份文件来自兼容的豆匣版本 |
